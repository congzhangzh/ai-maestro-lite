# 产品背景

## 为什么要做这个产品

企业内部使用 AI 工具时，最大的成本通常不是“模型能不能用”，而是下面这些事情：

- 工具太多，用户找不到统一入口
- 安装和配置步骤太多
- 代理、镜像源、模型网关各自分散
- 不同场景下该装什么、怎么启动，用户并不清楚
- 更新、下载、访问审计又有企业内部要求

所以这个产品的目标不是做一个什么都能编排的平台，而是做一个简单、固定、能落地的企业桌面助手。

## 产品定位

AI Maestro Lite 的定位很明确：

- 面向企业内部
- 面向固定业务场景
- 提供统一桌面入口
- 提供统一后端配置和审计能力

它不是通用平台，也不是低代码流程引擎。

## 固定场景

当前只做 5 个固定场景：

- `VS Code` 编程开发
- `Claude Code` 编程开发
- `Claude Code` 写 PPT
- `百度 Comate` 编程
- `受控 GitHub 浏览器`

## 核心原则

- 场景优先，不做抽象过度的“能力平台”
- 用户看到的动作必须简单：安装、更新、启动、自检
- 所有更新都先展示说明，再让用户确认
- 审计和中台同步是内建能力，不是后加功能
- 默认即可运行，同时允许企业配置覆盖

## 为什么强调简单

这个产品的使用者不是为了研究框架，而是为了尽快把环境准备好。

因此在技术和产品设计上都刻意收敛：

- 后端用 `Express`，因为更直白
- 数据库用 `SQLite`，因为部署简单
- 场景固定，不引入通用编排概念
- 更新信息和下载地址支持“默认值 + 配置覆盖”，减少环境准备成本
