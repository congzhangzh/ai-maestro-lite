# 产品背景

## 为什么做这个产品

企业内部使用 AI 工具时，真正的门槛往往不是“有没有模型”，而是：

- 工具太多，入口分散
- 配置太多，代理、镜像源、模型网关、IDE 插件各自一套
- 用户不知道当前机器缺什么、该先装什么
- 不同业务场景的操作步骤不一样，噪声很大
- 合规与审计要求又使浏览、下载、更新不能完全放开

因此，产品目标不是做一个“万能 AI 平台”，而是做一个**场景化的企业桌面助手**，把安装、配置、启动、自检、更新确认和审计统一起来。

## 产品定位

AI Maestro Lite 的定位是：

- 企业内部 AI 工具场景助手
- 桌面端统一入口
- 后端提供配置、更新、探针和审计服务
- 面向固定业务场景，不做通用编排引擎

## 当前固定业务场景

- VS Code 编程开发
- Claude Code 编程开发
- Claude Code 写 PPT
- 百度 Comate 编程
- 受控 GitHub 浏览器

## 核心原则

- 业务场景优先，不追求抽象通用性
- 用户看到的是简单动作：安装、更新、启动、自检
- 所有更新都先展示说明，再让用户确认
- 关键动作与访问行为都可审计
- 默认可运行，同时允许企业侧配置覆盖
